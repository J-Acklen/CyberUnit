What is a PCAP File?

PCAP stands for Packet Capture

  • A binary file format that stores network packet data
  • Contains raw network traffic including headers and payloads
  • Can be analyzed offline long after traffic was captured
  • Standard format supported by multiple tools

Essential Analysis Tools

Wireshark

The industry-standard GUI tool for packet analysis

  • Powerful display filters and search capabilities
  • "Follow Stream" feature for reconstructing conversations
  • Protocol dissectors for hundreds of protocols
  • Export capabilities for specific data

TShark

Command-line version of Wireshark for automation

# Basic capture viewing
tshark -r traffic.pcap

# Filter HTTP traffic
tshark -r traffic.pcap -Y "http"

Key Analysis Concepts

Understanding the OSI Model

  • Layer 2 (Data Link): MAC addresses, switches
  • Layer 3 (Network): IP addresses, routing
  • Layer 4 (Transport): TCP/UDP ports, connections
  • Layer 7 (Application): HTTP, FTP, DNS, etc.

Technique: Following TCP Streams

The "Follow TCP Stream" feature reconstructs entire conversations

How to Use It:

  • Right-click any packet in the conversation
  • Select "Follow" ‚Üí "TCP Stream"
  • View the complete back-and-forth exchange
  • Color coding shows client vs. server traffic
What You Can Find:
  • Credentials: Usernames and passwords in plain text protocols
  • Commands: System commands executed remotely
  • File contents: Text files transferred via FTP or HTTP
  • API requests: Full HTTP requests and responses

This is one of the most powerful features for understanding what actually happened!

Advanced: Identifying Malicious Patterns

After initial compromise, attackers establish persistent access and exfiltrate data

Common Attack Indicators:

  • Initial Access: Downloads from suspicious domains
  • C2 Communication: Periodic beaconing to external IPs
  • Lateral Movement: SMB or RDP connections to internal hosts
  • Data Exfiltration: Large outbound transfers
Wireshark Filters for Investigation:
# Find DNS queries
dns.qry.name contains "malicious"

# HTTP downloads
http.request.method == "GET" && http.response

# External connections from internal host
ip.src == 10.1.17.0/24 && !(ip.dst == 10.0.0.0/8)

  • Check hex dumps for readable text in binary protocols

Export Objects

File ‚Üí Export Objects ‚Üí HTTP/SMB/FTP to extract files transferred over the network

Security Red Flags in PCAP Analysis

  • Unencrypted protocols: Telnet, FTP, HTTP transmitting sensitive data
  • Unusual ports: Common services on non-standard ports
  • Failed login attempts: Multiple authentication failures followed by success
  • Suspicious file transfers: Executables, shells, or exploits
  • Command execution: Web requests with system commands in parameters
  • Data exfiltration: Large outbound transfers to external IPs
Pro Tip:
Always check "Statistics" ‚Üí "Conversations" in Wireshark to identify the most active connections and unusual communication patterns.
